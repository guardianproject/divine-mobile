# Camera "Upload" mode explainer — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a third `Upload` tab to the camera mode-switcher that opens an in-screen explainer (no picker) about why Divine doesn't accept camera-roll uploads, with a single outbound link to `divine.video/proofmode`.

**Architecture:** New enum case `VideoRecorderMode.upload` rendered by the existing horizontal mode wheel. A new `VideoRecorderUploadStack` swaps in for the camera/record stacks when `upload` is the active mode. The recorder screen reuses the existing `handleAppLifecycleState(.paused)` path to release the camera while the explainer is showing, and re-initializes when the user swipes back to `capture`/`classic`. One outbound URL via `url_launcher` (already in `pubspec.yaml`).

**Tech Stack:** Flutter, Riverpod, `url_launcher: ^6.3.2` (existing), `divine_ui` design tokens, `flutter_localizations` / ARB.

**Spec:** `docs/superpowers/specs/2026-05-04-camera-upload-mode-explainer-design.md`

  final bool useExternalApplication;

class UrlLauncherTestDouble extends UrlLauncherPlatform {
  final List<_LaunchedCall> launched = [];

```dart
void setRecorderMode(
  VideoRecorderMode mode, {
  bool keepAutosavedDraft = false,
}) {
  final previousMode = state.recorderMode;
  state = state.copyWith(
    recorderMode: mode,
    aspectRatio: mode.defaultAspectRatio,
    showGridLines: mode.supportGridLines,
  );
  final prefs = ref.read(sharedPreferencesProvider);
  prefs.setString(kLastUsedRecorderModeKey, mode.name);

  // Upload mode is an explainer-only tab; it doesn't own clips or
  // editor state. Preserve both when entering or leaving upload so the
  // user doesn't lose work for tapping a curious tab.
  final touchesRecordingState =
      mode != VideoRecorderMode.upload &&
      previousMode != VideoRecorderMode.upload;
  if (touchesRecordingState) {
    ref
        .read(clipManagerProvider.notifier)
        .clearAll(keepAutosavedDraft: keepAutosavedDraft);
    ref
        .read(videoEditorProvider.notifier)
        .reset(keepAutosavedDraft: keepAutosavedDraft);
  }

  Log.debug(
    '🎬 Recorder mode changed to: ${mode.name}',
    name: 'VideoRecorderNotifier',
    category: .video,
  );
}
```
